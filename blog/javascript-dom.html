HTML, CSS, JavaScript & the DOM

Think of a web page like building a level in a game engine: HTML is the level geometry/meshes (walls, floors, props), CSS is the materials, shaders and lighting (how it looks), JavaScript is your Blueprints/scripts (what makes things react), and the DOM is the scene hierarchy — the live tree where every object sits so your scripts can find and change them.

What is the DOM (in game-dev terms)?

The Document Object Model is like the outliner/scene tree in Unreal or Unity — a tree of all actors on the level. Each heading, button or image is a node. JavaScript uses selectors (like searching the outliner) to grab a node and tweak it.

Mini-demo: change a heading

Loading screen: Press Start

// Select a node from the DOM (scene tree)
const title = document.querySelector('#sceneTitle')
// Update its text at runtime
title.textContent = "Game Started — Let's Go!"

Control Flow = The Training Plan

Control flow is just the order your coach gives instructions. You follow them step by step. Sometimes you take a different path depending on what happens (like “if you're tired, take a break”).

Example: MMA Training Flow

  • Warm up.
  • If gloves are on → start striking drills. Otherwise → put on gloves first.
  • Do combos until time is up.
  • Cool down and stretch.

That's control flow: a set of decisions and steps in order.

Loops = Repeated Drills

Loops are the reps or rounds you do over and over.

Example: “Throw 10 jabs” = a loop that repeats 10 times.

let glovesOn = false;
let rounds = 3;

// Step 1: Warm up
console.log("Jog around the gym/shadowbox");

// Step 2: If/else (decision making)
if (!glovesOn) {
  console.log("Put on gloves");
  glovesOn = true;
}
console.log("Start striking drills");

// Step 3: Loop (repeat combos for 3 rounds)
for (let round = 1; round <= rounds; round++) {
  console.log("Round " + round + ": Throw combo (jab → cross → hook)");
}

// Step 4: Finish up
console.log("Cool down and stretch");

Breaking it Down

  • Control flow: warm up → gloves check → drills → cooldown. If/else decides what happens (gloves on or not).
  • Loop: the for loop repeats the combo each round. Instead of writing “Throw combo” three times, the loop handles it.

Arrays vs Objects

Arrays = Ordered Lists

Think of an array like a line of fighters waiting to spar.

Each fighter has a number (their position in line), starting at 0. You use the number (called an index) to grab the data. 

let fighters = ["Adesanya", "Hooker", "Volkanovski"];

console.log(fighters[0]); // Adesanya (first in line)
console.log(fighters[1]); // Hooker
console.log(fighters[2]); // Volkanovski

How you access data: array[index] (always a number). Order matters.

Objects = Named Lockers

An object is more like a locker room where every locker has a label (a key).

Instead of remembering “Locker #2,” you just say “Hooker's locker.”

let fighterStats = {
  name: "Hooker",
  style: "Kickboxing, Brazilian Jui Jitsu",
  wins: 24,
  losses: 12,
  weight: "155lbs"
};

console.log(fighterStats.name);     // Hooker
console.log(fighterStats["style"]); // Kickboxing, Brazilian Jui Jitsu

How you access data: object.key or object["key"]. Keys are like names, not numbers. Order doesn't matter — you just call the key.

Side-by-Side

Feature Array Object
Structure Ordered list Key-value pairs
Access by Index (number) Key (string)
Example fighters[0] → "Adesanya" fighterStats.style → "Kick-Boxing"
Best for Lists, sequences, ordered data Profiles, settings, grouped data

So..

  • Use an array when you need an ordered list (like a roster).
  • Use an object when you need to group related data (like a fighter's stats).
  • Sometimes you use both together (array of objects).

Functions — MMA Analogy

Think of a function like a fight combo.

In training, you practice: jab → cross → hook → low kick.

Instead of saying all those moves each time, your coach just says: “Combo One!” — you instantly know the steps and do them.

That's a function in JS: you give it a name (comboOne) and when you “call” it, the fighter executes all the moves inside. 

function comboOne() {
  return "jab → cross → hook → low kick"
}

Why it Helps

  • You don't have to write every strike in your code each time. Wrap it up in a function and call it whenever you need. Like a shortcut.
  • If you want to change the combo (swap the hook for an uppercut), you update it once, then every time you call (comboOne), the new combo is used.
  • Functions help organize code into logical chunks. Each function does one thing (like a specific combo). This makes your code easier to read and maintain.

Conclusion

HTML is the skeleton, CSS dresses it up, JavaScript makes it functional ⚡, and the DOM is the translator that lets JavaScript communicate with HTML.
